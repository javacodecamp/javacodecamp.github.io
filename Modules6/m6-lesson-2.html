Lesson 6.2: Auto-boxing နှင့် Unboxing

ကြာမြင့်ချိန်: 10 မိနစ် | အဆင့်: အခြေခံ

၁။ Auto-boxing ဆိုသည်မှာ ဘာလဲ?

Auto-boxing ဆိုသည်မှာ Java Compiler က Primitive data types များကို ၎င်းတို့နှင့် သက်ဆိုင်ရာ Wrapper class objects များအဖြစ် အလိုအလျောက် ပြောင်းလဲပေးခြင်း ဖြစ်သည်။ ဥပမာ- int မှ Integer သို့ ပြောင်းခြင်း။

၂။ Unboxing ဆိုသည်မှာ ဘာလဲ?

Unboxing ဆိုသည်မှာ Wrapper class object တစ်ခုမှ ၎င်း၏ သက်ဆိုင်ရာ Primitive data type သို့ အလိုအလျောက် ပြန်လည်ပြောင်းလဲပေးခြင်း ဖြစ်သည်။ ဥပမာ- Integer မှ int သို့ ပြောင်းခြင်း။

Auto-boxing နှင့် Unboxing ဥပမာ


public class Main {
    public static void main(String[] args) {
        // ၁။ Auto-boxing (int to Integer)
        int a = 20;
        Integer obj = a; // Compiler က Integer.valueOf(a) ကို အလိုအလျောက် လုပ်ပေးသည်

        // ၂။ Unboxing (Integer to int)
        Integer b = 50;
        int c = b; // Compiler က b.intValue() ကို အလိုအလျောက် လုပ်ပေးသည်

        System.out.println("Object value: " + obj);
        System.out.println("Primitive value: " + c);
    }
}

Output ရလဒ်

Object value: 20
Primitive value: 50

၃။ အဘယ်ကြောင့် အသုံးပြုသနည်း?

Auto-boxing နှင့် Unboxing ကြောင့် Developer များသည် Primitive နှင့် Object ကြား ပြောင်းလဲရန် Method များကို ကိုယ်တိုင် ခေါ်ယူရေးသားရန် မလိုတော့ဘဲ Code များကို ပိုမိုရှင်းလင်းစွာ ရေးသားနိုင်လာသည်။ အထူးသဖြင့် **Collections (ArrayList, etc.)** များတွင် Primitive များကို အသုံးပြုသည့်အခါ အလွန်အသုံးဝင်သည်။

**သတိထားရန်:** Auto-boxing သည် အလုပ်လုပ်ရာတွင် အချိန်အနည်းငယ် ပိုယူတတ်သဖြင့် အကြိမ်ရေ သန်းပေါင်းများစွာ လည်ပတ်နေသော Loop များအတွင်း၌ Primitive types များကိုသာ အသုံးပြုခြင်းက Performance ပိုကောင်းစေပါသည်။

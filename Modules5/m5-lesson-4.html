Lesson 5.4: Encapsulation (အချက်အလက်ဖုံးကွယ်ခြင်း)

ကြာမြင့်ချိန်: 20 မိနစ် | အဆင့်: အလယ်အလတ်

၁။ Encapsulation ဆိုသည်မှာ ဘာလဲ?

Encapsulation ဆိုသည်မှာ Variables (Data) များနှင့် ၎င်းတို့ကို အလုပ်လုပ်သော Methods များကို စုစည်းပြီး တစ်ခုတည်းသော Unit (Class) အတွင်း ထည့်သွင်းထားခြင်း ဖြစ်သည်။ အချက်အလက်များကို ပြင်ပမှ တိုက်ရိုက် မမြင်နိုင်ရန်နှင့် မပြင်ဆင်နိုင်ရန် Private အဖြစ် ထားရှိပြီး စစ်ဆေးမှုများ ပြုလုပ်နိုင်ရန် Getters နှင့် Setters တို့ကို အသုံးပြုသည်။

၂။ Encapsulation ပြုလုပ်ပုံ အဆင့်ဆင့်

  1. Class အတွင်းရှိ Variables များကို private အဖြစ် ကြေညာပါ။
  2. တန်ဖိုးကို ကြည့်ရှုရန်အတွက် public get...() method ဖန်တီးပါ။
  3. တန်ဖိုးကို ပြင်ဆင်ရန်အတွက် public set...() method ဖန်တီးပါ။

Encapsulation ဥပမာ (Student Class)


class Student {
    // ၁။ Private Variables (ပြင်ပက တိုက်ရိုက် မမြင်ရပါ)
    private String name;
    private int age;

    // ၂။ Getter Method (တန်ဖိုး ထုတ်ပေးရန်)
    public String getName() {
        return name;
    }

    // ၃။ Setter Method (တန်ဖိုး ထည့်သွင်းရန်)
    public void setName(String newName) {
        this.name = newName;
    }

    // Setter တွင် Logic များ ထည့်သွင်း စစ်ဆေးနိုင်သည်
    public void setAge(int newAge) {
        if (newAge > 0) {
            this.age = newAge;
        } else {
            System.out.println("Error: အသက်သည် ၀ ထက် ကြီးရမည်။");
        }
    }

    public int getAge() {
        return age;
    }
}

public class Main {
    public static void main(String[] args) {
        Student s = new Student();
        
        // s.name = "Mg Mg"; // Error တက်မည် (Private ဖြစ်သောကြောင့်)
        
        s.setName("Mg Mg");
        s.setAge(20);

        System.out.println("နာမည်: " + s.getName());
        System.out.println("အသက်: " + s.getAge());
    }
}

Output ရလဒ်

နာမည်: Mg Mg
အသက်: 20

၃။ အဘယ်ကြောင့် Encapsulation ကို သုံးရသနည်း?

  • **Better Control:** Variable တန်ဖိုးများကို Setter ထဲတွင် စစ်ဆေးပြီးမှ ပြောင်းလဲခွင့် ပေးနိုင်သည်။ (ဥပမာ- အသက်ကို အနှုတ်ကိန်း ပေး၍မရအောင် တားဆီးခြင်း)။
  • **Security:** အရေးကြီးသော အချက်အလက်များကို တိုက်ရိုက် မပြင်နိုင်အောင် ဖုံးကွယ်ထားနိုင်သည်။
  • **Flexibility:** စာမျက်နှာပြင်ပက Code များကို မထိခိုက်ဘဲ Class အတွင်းရှိ Logic များကို စိတ်ကြိုက် ပြောင်းလဲနိုင်သည်။
Encapsulation ကို အလွယ်မှတ်ရလျှင် **"Private Fields + Public Getters/Setters"** ဟု မှတ်သားနိုင်ပါသည်။

Lesson 5.3: Polymorphism (ပုံသဏ္ဌာန်အမျိုးမျိုးပြောင်းလဲခြင်း)

ကြာမြင့်ချိန်: 35 မိနစ် | အဆင့်: အလယ်အလတ်

၁။ Polymorphism ဆိုသည်မှာ ဘာလဲ?

Polymorphism သည် ဂရိစကားလုံးဖြစ်ပြီး "ပုံသဏ္ဌာန်များစွာ" ဟု အဓိပ္ပာယ်ရသည်။ OOP တွင်မူ Action တစ်ခုတည်းကို မတူညီသော ပုံစံများဖြင့် လုပ်ဆောင်နိုင်စွမ်းကို ဆိုလိုသည်။ ၎င်းကို အမျိုးအစား နှစ်မျိုး ခွဲခြားနိုင်သည်။

၂။ Compile-time Polymorphism (Static Binding)

၎င်းကို **Method Overloading** ဟုလည်း ခေါ်သည်။ Program ကို Compile လုပ်နေစဉ်အတွင်း မည်သည့် Method ကို ခေါ်ယူမည်ကို ဆုံးဖြတ်ခြင်း ဖြစ်သည်။ (Module 4 တွင် ရှင်းပြခဲ့သည့်အတိုင်း Parameter မတူညီသော နာမည်တူ Method များ ဖြစ်သည်)။

၃။ Runtime Polymorphism (Dynamic Binding)

၎င်းကို **Method Overriding** ဟု ခေါ်သည်။ အမွေဆက်ခံထားသော (Inheritance) Class များတွင် မိဘ Class ရှိ Method ကို သားသမီး Class က မိမိစိတ်ကြိုက် ပြန်လည်ပြင်ဆင် ရေးသားခြင်း ဖြစ်သည်။ မည်သည့် Method အလုပ်လုပ်မည်ကို Program Run နေသည့်အချိန်မှသာ ဆုံးဖြတ်သည်။

Method Overriding ဥပမာ


class Animal {
    void makeSound() {
        System.out.println("တိရစ္ဆာန်လေးတွေ အော်သံပြုကြသည်...");
    }
}

class Dog extends Animal {
    @Override
    void makeSound() {
        System.out.println("ခွေးလေးက: ဝုတ် ဝုတ်!");
    }
}

class Cat extends Animal {
    @Override
    void makeSound() {
        System.out.println("ကြောင်လေးက: မြောင် မြောင်!");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal myAnimal = new Animal();
        Animal myDog = new Dog(); // Polymorphic object
        Animal myCat = new Cat();

        myAnimal.makeSound();
        myDog.makeSound();  // Runtime တွင် Dog ၏ method ကို ခေါ်မည်
        myCat.makeSound();  // Runtime တွင် Cat ၏ method ကို ခေါ်မည်
    }
}

Output ရလဒ်

တိရစ္ဆာန်လေးတွေ အော်သံပြုကြသည်...
ခွေးလေးက: ဝုတ် ဝုတ်!
ကြောင်လေးက: မြောင် မြောင်!

၄။ အဘယ်ကြောင့် Polymorphism ကို သုံးရသနည်း?

  • **Flexibility:** Object တစ်ခုတည်းက အခြေအနေပေါ်မူတည်ပြီး ပုံစံအမျိုးမျိုး ပြောင်းလဲလုပ်ဆောင်နိုင်သည်။
  • **Clean Code:** Method နာမည် တစ်ခုတည်းကိုသာ မှတ်သားထားရန် လိုအပ်ပြီး အလုပ်လုပ်ပုံကို Object က ဆုံးဖြတ်ပေးသည်။
**@Override** ဆိုသည်မှာ Annotation တစ်ခု ဖြစ်သည်။ ၎င်းသည် မိဘဆီမှ Method ကို ပြန်ပြင်ရေးနေခြင်းဖြစ်ကြောင်း Compiler ကို အသိပေးခြင်းဖြစ်ပြီး မှားယွင်းမှုများကို ကာကွယ်ပေးသည်။

Lesson 5.5: Abstraction (အနှစ်ထုတ်ခြင်း)

ကြာမြင့်ချိန်: 20 မိနစ် | အဆင့်: အလယ်အလတ်

၁။ Abstraction ဆိုသည်မှာ ဘာလဲ?

Abstraction ဆိုသည်မှာ အရာဝတ္ထုတစ်ခု၏ အရေးကြီးသော လက္ခဏာရပ်များကိုသာ ပြသပြီး နောက်ကွယ်မှ ရှုပ်ထွေးသော အလုပ်လုပ်ပုံများကို ဖုံးကွယ်ထားခြင်း ဖြစ်သည်။ Java တွင် Abstraction ကို နည်းလမ်းနှစ်မျိုးဖြင့် ရရှိနိုင်သည်-

  • Abstract Class (၀ မှ ၁၀၀ ရာခိုင်နှုန်းအထိ Abstraction ပေးနိုင်သည်)
  • Interface (၁၀၀ ရာခိုင်နှုန်း အပြည့်အဝ Abstraction ပေးနိုင်သည်)

၂။ Abstract Class နှင့် Abstract Method

Abstract Class: ၎င်းကို Object အဖြစ် တိုက်ရိုက်ဖန်တီး၍ မရပါ။ အခြား Class များက အမွေဆက်ခံရန်အတွက်သာ အသုံးပြုသည်။

Abstract Method: ၎င်းတွင် Body (Code အလုပ်လုပ်ပုံ) မရှိပါ။ အမည်နှင့် ပုံစံကိုသာ သတ်မှတ်ထားပြီး အမွေဆက်ခံသော Subclass များက မဖြစ်မနေ ပြန်လည်ရေးသား (Override) ရမည်ဖြစ်သည်။

Abstraction ဥပမာ (Shape Class)


// Abstract Class တစ်ခု ကြေညာခြင်း
abstract class Shape {
    String color;

    // Abstract Method (Body မပါဝင်ပါ)
    abstract double calculateArea();

    // ပုံမှန် Method လည်း ပါဝင်နိုင်သည်
    void display() {
        System.out.println("ဤအရာသည် ပုံသဏ္ဍာန်တစ်ခု ဖြစ်သည်။");
    }
}

// Subclass က အမွေဆက်ခံပြီး abstract method ကို ဖြည့်စွက်ခြင်း
class Circle extends Shape {
    double radius;

    Circle(double r) { this.radius = r; }

    @Override
    double calculateArea() {
        return Math.PI * radius * radius;
    }
}

public class Main {
    public static void main(String[] args) {
        // Shape s = new Shape(); // Error: Abstract class ကို object ဆောက်၍မရပါ
        
        Circle myCircle = new Circle(5.0);
        myCircle.display();
        System.out.println("စက်ဝိုင်း၏ ဧရိယာ: " + myCircle.calculateArea());
    }
}

Output ရလဒ်

ဤအရာသည် ပုံသဏ္ဍာန်တစ်ခု ဖြစ်သည်။
စက်ဝိုင်း၏ ဧရိယာ: 78.53981633974483

၃။ အဘယ်ကြောင့် Abstraction ကို သုံးရသနည်း?

  • **Security:** အသုံးပြုသူကို လိုအပ်တဲ့ အစိတ်အပိုင်းတွေကိုပဲ ပြသပြီး အဓိက Code တွေကို ဖုံးကွယ်ထားနိုင်ခြင်း။
  • **Simplicity:** ရှုပ်ထွေးမှုများကို လျှော့ချပေးပြီး Program ကို ပိုမိုရှင်းလင်းစေခြင်း။
  • **Standardization:** Subclass အားလုံးမှာ တူညီတဲ့ Method နာမည်တွေ ပါဝင်လာအောင် ပုံစံခွက် ချပေးနိုင်ခြင်း။
Abstract Class တစ်ခုထဲမှာ အနည်းဆုံး Abstract Method တစ်ခုပါဝင်နေရင် အဲ့ဒီ Class ကိုလည်း `abstract` လို့ မဖြစ်မနေ ကြေညာပေးရပါတယ်။

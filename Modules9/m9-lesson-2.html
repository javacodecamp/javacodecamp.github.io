Lesson 9.2: Checked vs Unchecked နှင့် throw, throws

ကြာမြင့်ချိန်: 18 မိနစ် | အဆင့်: အလယ်အလတ်

၁။ Checked vs Unchecked Exceptions

Java တွင် Exception များကို Compile လုပ်ချိန်၌ စစ်ဆေးမှု ရှိ/မရှိ ပေါ်မူတည်၍ နှစ်မျိုးခွဲခြားထားသည်။

Checked Exception Unchecked Exception
Compile-time တွင် စစ်ဆေးသည်။ Runtime (Run နေစဉ်) မှသာ ဖြစ်ပေါ်သည်။
မဖြစ်မနေ try-catch သို့မဟုတ် throws သုံးရသည်။ အသုံးပြုရန် မဖြစ်မနေ မလိုအပ်ပါ။
ဥပမာ - IOException, SQLException ဥပမာ - NullPointerException, ArrayIndexOutOfBoundsException

၂။ `throw` Keyword

throw keyword ကို Exception တစ်ခုကို ကိုယ်တိုင် **ပစ်ထုတ် (Manually create)** ရန်အတွက် အသုံးပြုသည်။

throw အသုံးပြုပုံ ဥပမာ


static void validateAge(int age) {
    if (age < 18) {
        // Exception တစ်ခုကို ကိုယ်တိုင် ပစ်ထုတ်လိုက်သည်
        throw new ArithmeticException("အသက် ၁၈ မပြည့်သဖြင့် မဲပေးခွင့်မရှိပါ။");
    } else {
        System.out.println("မဲပေးနိုင်ပါသည်။");
    }
}

၃။ `throws` Keyword

throws keyword ကို Method Signature တွင် အသုံးပြုပြီး ၎င်း Method အတွင်းမှ တက်လာနိုင်သော Exception များကို **ကြိုတင်အသိပေး (Declare)** ရန် သုံးသည်။ ၎င်းသည် Error ကို မဖြေရှင်းဘဲ ခေါ်ယူအသုံးပြုမည့်သူထံ တာဝန်လွှဲလိုက်ခြင်း ဖြစ်သည်။

throws အသုံးပြုပုံ ဥပမာ


import java.io.*;

class Test {
    // ဤ method သည် IOException တက်နိုင်သည်ဟု ကြေညာထားသည်
    void readFile() throws IOException {
        throw new IOException("ဖိုင်ဖတ်၍မရပါ။");
    }
}

public class Main {
    public static void main(String[] args) {
        Test obj = new Test();
        try {
            obj.readFile();
        } catch (IOException e) {
            System.out.println("Error ကို ဤနေရာတွင် ဖမ်းလိုက်သည်: " + e.getMessage());
        }
    }
}
**မှတ်သားရန်:** `throw` ကို Method ထဲတွင် အသုံးပြုပြီး `throws` ကို Method နာမည်ဘေးတွင် အသုံးပြုသည်။
